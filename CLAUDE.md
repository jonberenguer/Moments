## Project Overview

**Moments** is a cross-platform desktop photo & video slideshow editor built with Electron. Users import media, arrange clips on a timeline, apply transitions and text overlays, and export a finished MP4. Video encoding runs via a **native FFmpeg child process** with GPU acceleration — replacing the original browser-based WebAssembly engine entirely.

**Platforms:** Linux (x64) · Windows (x64)

**Stack**

| Layer | Tool |
|---|---|
| Shell | Electron 35 |
| Framework | React 19 + Vite 8 |
| Video encoding | Native FFmpeg binary (bundled) via IPC child process |
| GPU acceleration | NVENC (NVIDIA) → AMF (AMD) → QSV (Intel iGPU) → CPU |
| Icons | `lucide-react` |
| Styling | CSS Modules |
| State | Custom hook (`useMediaStore`) — no external state lib |

**Run locally**
```bash
npm install
# Place FFmpeg binary at bin/linux/ffmpeg (Linux) or bin/win/ffmpeg.exe (Windows)
npm run dev   # starts Vite on :5173 + Electron
```

## ⚠️ UI Global Variables — Required for Export

These values all live in `useMediaStore` and are read directly from `storeRef.current` inside `handleFFmpegLoad` in `App.jsx`. They must never be assumed, hardcoded, or passed via intermediate refs — always read from store at the moment export fires.

| Variable | Source | Used for |
|---|---|---|
| `quality` | Topbar toggle (`480p` / `720p` / `1080p`) | `QUALITY_DIMS[quality][aspectRatio]` → canvas `W × H` |
| `aspectRatio` | Topbar toggle (`16:9` / `9:16`) | Canvas orientation + xfade transitions |
| `musicVolume` | Inspector Music section slider (0–100) | Normalised to 0.0–1.0 for FFmpeg `volume=` filter |
| `musicTrimStart` / `musicTrimEnd` | Inspector Music Trim bar | Music segment selection |
| `exportDuration` | `useMediaStore` — computed from all clips with trim/speed/transitions applied | Master timeline length displayed in Timeline chip |
| `endFadeVideo` / `endFadeVideoDuration` | Inspector End Fade section | Fade-to-black filter at end of timeline |
| `endFadeAudio` / `endFadeAudioDuration` | Inspector End Fade section | Audio fade-out filter at end of timeline |

### Rules

- `quality`, `musicVolume`, and all other export-affecting state live in **`useMediaStore`** — not in `App.jsx`. This ensures they are persisted in saved workflows and always consistent with the export.
- `W` and `H` (canvas dimensions) are derived once at the top of `exportMoment()` as `QUALITY_DIMS[quality][aspectRatio]` and passed down to every filter builder. They are always even integers. Never recompute or hardcode them inside a filter string.

## ⚠️ Export Pipeline — DO NOT BREAK

**The export pipeline is extremely sensitive to changes.** Any modification to `useFFmpeg.js` must preserve all of the following invariants or the export will produce corrupt output or fail silently.

### Threading

Native FFmpeg has no threading restrictions. GPU encoders manage their own parallelism internally. For the CPU fallback (`libx264`), `-threads 0` is used to allow FFmpeg to use all available cores.

### Temp directory lifecycle — CRITICAL

Source files, intermediate segments, and the final output all live in an OS temp directory created at the start of each export (`os.tmpdir()/moments_export_XXXXX`).

**The main process (`electron/main.js`) must NOT delete this directory in its `cleanup()` function.** The renderer reads the output file via `api.readFile()` after `startExport()` resolves — deleting the temp dir in main's cleanup races with that read and causes ENOENT.

The renderer's `finally` block owns cleanup: it reads the output, copies it to the user's chosen save path, then calls `api.rmdir(tmpDir)`.

```
Renderer finally block order (MUST NOT change):
  1. api.readFile(outputFile)     ← read output from temp dir
  2. api.saveFileDialog()         ← user picks save location
  3. api.copyFile(tmp, savePath)  ← copy to final destination
  4. cleanup() → api.rmdir(tmpDir) ← NOW safe to delete
```

### Audio stream fallback — CRITICAL

Video clips without an audio stream must not cause the export to fail. The audio extraction step uses `-map 0:a:0` (explicit first audio stream). If no audio stream exists, FFmpeg exits non-zero and the main process automatically runs the `fallbackOnFail` silence step.

### Encoder tokens

The renderer builds FFmpeg argument arrays with two special tokens that main.js resolves at execution time:

- `__ENCODER__` → replaced with the selected HW encoder string (e.g. `h264_nvenc`, `libx264`)
- `__ENC_ARGS__` → spread-replaced with the encoder's quality arg array

### `IMAGE_EXTS` must be declared inside `exportMoment()`

```js
// Required at the top of exportMoment() try block:
const IMAGE_EXTS = new Set(['jpg','jpeg','png','gif','webp','heic','avif'])
```

### Font path escaping on Windows — CRITICAL

FFmpeg's `drawtext` filter uses `:` as its option separator. Windows drive letter colons (e.g. `C:/path`) must be escaped as `C\:/path` — **single quotes do NOT protect colons** in FFmpeg filter syntax. This is handled in `buildDrawtext`:

```js
const escapedPath = fontPath
  .replace(/\\/g, '/')            // backslash → forward slash
  .replace(/^([A-Za-z]):/, '$1\\:') // C: → C\:
  .replace(/'/g, "\\'")           // escape single quotes
```

Do not remove or simplify this escaping — it will silently break text overlay export on Windows.

### Source file index consistency — CRITICAL

When building `validClips`, each clip's associated temp files (`src_N`, `aud_N`, `seg_N`) must all use the same index `N` — the clip's position in the `validClips` array, **not** its position in the original `clips` array. Clips with missing files are skipped, so using the original index creates a mismatch.

```js
const vi    = validClips.length   // index BEFORE push
const fname = `src_${vi}.${ext}`
validClips.push({ ...c, _fname: fname, ... })
```

### Font handling — native paths

Fonts are referenced by real OS paths resolved via `api.fontPath(fontFile)`. There is no WASM FS write step. Custom font data (from `seg.customFontData`) is written to the temp dir as `font_custom.ttf` before the step list is sent to main.

Custom font bytes must still be `.slice()`'d when stored into React state to avoid detached ArrayBuffer errors on repeated exports.

### Detection

On each export open, the main process runs `ffmpeg -encoders` to check compiled-in HW encoders, then smoke-tests each with a 1-frame encode. Results are cached per session.

### Priority

| Priority | Encoder | Hardware |
|---|---|---|
| 1 | `h264_nvenc` | NVIDIA GPU |
| 2 | `h264_amf` | AMD GPU |
| 3 | `h264_qsv` | Intel iGPU |
| 4 | `h264_v4l2m2m` | Linux V4L2 |
| 5 | `libx264` | CPU (always available) |

### Encoder quality args

## IPC Architecture

```
Renderer (React)
  └─ window.electronAPI.*          (injected by preload.js via contextBridge)
        ↓
  Electron Main Process (electron/main.js)
  ├─ gpu:detect       → detectGPUCapabilities() → smoke-tests each encoder
  ├─ gpu:reset        → clears cached GPU caps
  ├─ ffmpeg:export    → resolves encoder tokens, spawns FFmpeg, streams logs back
  ├─ ffmpeg:cancel    → kills active FFmpeg process
  ├─ fs:readFile      → returns base64-encoded file contents
  ├─ fs:writeFile     → writes base64-encoded data to path
  ├─ fs:copyFile      → copies file to save path
  ├─ fs:deleteFile    → deletes file
  ├─ fs:mkdtemp       → creates temp directory, returns path
  ├─ fs:rmdir         → recursively removes directory
  ├─ fs:fontPath      → resolves bundled font path (resources/ in production)
  ├─ dialog:saveFile  → shows native OS save dialog
  ├─ dialog:openFiles → shows native OS open dialog, remembers last folder via prefs
  ├─ prefs:get        → reads a key from persistent prefs.json in userData dir
  ├─ prefs:set        → writes a key to persistent prefs.json
  └─ shell:openPath   → opens file in system default app
```


## Export Pipeline — `useFFmpeg.js` → `exportMoment()`

### Stage 1 — Per-clip encoding → `seg_N.mp4`

Every `seg_N.mp4` delivered to Stage 2 must conform exactly:

| Property | Value |
|---|---|
| Video codec | `__ENCODER__` (GPU or libx264) |
| Pixel format | yuv420p |
| Frame rate | 30 fps CFR |
| Timebase | 1/90000 |
| Audio codec | AAC 128k stereo 44100 Hz |

#### Image clip path

Single FFmpeg invocation using `filter_complex`: fit → even-align → pad → optional effect/EQ.

#### Image clip — blur background path

```
[0:v] split=2 → [_bg0][_fg0]
[_bg0] scale W:H (fill) → crop W:H → boxblur → [_bg]
[_fg0] scale W:H (fit, letterboxed, NO padding) → even-align → [_feven]
       optional eq → [_feven or _fpadded]
[_bg][_feven] overlay=(main_w-overlay_w)/2:(main_h-overlay_h)/2 → [vout]
```

- The fg is **never padded to W×H** after fitting. Padding makes `overlay_w = main_w` → offset collapses to `0:0` → blurred bg is hidden entirely.
- The fg stays at its natural letterboxed size so the blurred bg shows through the bars.
- `overlay` uses FFmpeg runtime vars (`main_w`, `main_h`, `overlay_w`, `overlay_h`) — not JS template vars.

#### Video clip path

Two steps: audio extraction (`-map 0:a:0` with silence fallback), then placement encode (fit → pad → speed/eq).

#### Video clip — blur background path

Same filter_complex strategy as image blur. Speed and eq are applied in the fg chain after the fit step.

### Stage 2 — Transitions, music → `prefinal.mp4`

Three paths:
- **Path A** — concat demuxer stream copy (all cuts, no music)
- **Path B** — single clip copy or music mix
- **Path C** — xfade/acrossfade filter_complex (mixed transitions or music)

### Stage 3 — Text overlays → `output.mp4`

All `drawtext` filters in one `-vf` pass. Font files referenced by real OS paths. Custom fonts written to temp dir before step list is sent. Windows drive letter colons escaped as `C\:/` — see Font path escaping section above.

Text timing uses `enable='between(t,startTime,startTime+duration)'`. The alpha expression drives fade animation for `animation === 'fade'` (and `'slide'`/`'typewriter'` as fallback — FFmpeg drawtext has no native slide or typewriter effect).

### Stage 4 — End fade → `output_final.mp4` (optional)

Applied only when `endFadeVideo` or `endFadeAudio` is true.

## Filter Builders (`useFFmpeg.js`)

| Function | Purpose |
|---|---|
| `buildEqFilter(b, c, s)` | Returns `eq=` filter string or `null` if all zero |
| `buildImageEffectVf(effect, W, H, duration)` | Returns zoompan `-vf` string or `null` |
| `buildDrawtext(seg, W, H, fontPath)` | Returns `drawtext=` filter string with proper Windows path escaping |
| `buildAtempoChain(speed)` | Returns `atempo` chain array for audio speed |

`buildVideoPlacementVf` has been removed — the fit+pad filter string is now inlined directly in the video clip path.

**Blur background filter_complex** (inline, not a named function):
- bg chain: `scale W:H (increase AR)` → `crop W:H` → `boxblur=luma_radius=20:luma_power=2`
- fg chain: `scale W:H (decrease AR, letterboxed, NO padding)` → `even-align` → optional eq
- composite: `overlay=(main_w-overlay_w)/2:(main_h-overlay_h)/2`
- **Never pad the fg to W×H** — that collapses the overlay offset to zero and hides the blur entirely.
- **Never use JS template vars (`W`, `H`) in the overlay expression** — use FFmpeg runtime vars (`main_w`, `main_h`, `overlay_w`, `overlay_h`).

## Known Gaps / Future Work

- **`fileDuration` for direct timeline drops** — when files are dragged directly to the timeline (bypassing the media panel), `fileDuration` is `null`. The TrimBar does not appear until the user previews the clip in the media panel first.
- **Speed + trim interaction** — `actualDur = (effectiveTrimEnd - trimStart) / speed`. Correct for wall-clock output duration.
- **Music file not saved in workflow** — the user must re-add the music file after loading a workflow. `musicTrimStart`, `musicTrimEnd`, and `musicVolume` are saved and restored correctly.
- **No undo/redo** — all mutations are direct `setState` calls.
- **Blur not rendered in Preview UI** — `blurBackground` is an export-only effect.
- **Per-clip transition duration** — stored but not yet applied in export (global `td` used for all transitions).
- **Multiple custom fonts** — only one custom font at a time (`font_custom.ttf`). Each segment stores its own `customFontData` so data is preserved, but the pipeline would need extending to write per-segment font files.
- **Media re-matching is filename-only** — on workflow load, clips are matched by filename alone.
- **`slide` and `typewriter` text animations** — these are CSS-only effects in the Preview. Both fall back to a fade in/out in the FFmpeg export, since `drawtext` has no native slide or typewriter support. A future approach would be to render text as an overlay video using `lavfi` or a separate compositing pass.
- **Brightness preview vs export discrepancy** — CSS `filter: brightness()` is multiplicative while FFmpeg `eq=brightness=` is an additive luma offset. The visual result differs at high values. A future fix would switch the export to `curves` or `lutyuv` for a true multiplicative match.
- **Contrast/saturation scale** — CSS uses `/100` divisor, FFmpeg uses `/50`. Export effect is approximately 2× stronger than the preview at any given slider value.

## Bug History (resolved)

### Text overlay timing
`buildDrawtext` was reading `seg.endTime` — a field never set anywhere. Fixed to derive `endTime = startTime + duration`. Text overlays now respect their configured duration.

### Text overlay animation
`alphaExpr` was gated on `seg.fadeIn` — a field never set anywhere, so all overlays exported with flat `alpha='1'`. Fixed to drive the fade expression from `seg.animation === 'fade'`.

### Source file index mismatch after workflow load
`validClips` was naming temp files using the original `clips` array index. When clips with missing files were skipped (common after workflow load), the `src_N`, `aud_N`, `seg_N` indices diverged, causing audio/video step mismatches. Fixed to use `validClips.length` before push as the consistent index.

### Blur background — fg covering blur entirely
Two bugs worked together: (1) the overlay expression used JS template vars `(W-w)/2` where `w` resolved to the overlay's own width, collapsing the offset to zero; (2) the fg was padded to `W×H` after fitting, making it the same size as the bg. Both fixed: overlay now uses `(main_w-overlay_w)/2:(main_h-overlay_h)/2`, and the pad step is removed so the fg stays at letterboxed size.

### Windows text overlay export failure
FFmpeg's `drawtext` filter uses `:` as its option separator. Windows drive letter colons (`C:/path`) were not escaped, causing the filter parser to split the path at the colon. Fixed by escaping `C:` → `C\:` in `buildDrawtext`.

### Windows build icon error
`package.json` pointed `win.icon` and `linux.icon` at `public/favicon.svg`. electron-builder's icon converter cannot process SVG. Fixed by generating `public/icon.png` (256×256 RGBA PNG from the SVG) and pointing both targets at it.

### Windows build NSIS uninstaller error
`win.target` was set to `nsis`. NSIS uninstaller stub generation fails when cross-compiling from Linux. Fixed by switching the default target to `portable`. The NSIS config is preserved as `_win_nsis` for native Windows builds.
